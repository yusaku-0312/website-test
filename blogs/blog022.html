【2026年最新】Google Workspace CLIとは？ターミナルからGmail・Drive・CalendarをAIと操作する革新ツール

記事の概要

Googleが公式GitHubリポジトリで公開したオープンソースツール「Google Workspace CLI（コマンド名：gws）」が話題を集めています。 Gmail・Drive・Calendar・Sheets・Docs・Chatなど、あらゆるWorkspaceサービスをターミナル1行で操作できるだけでなく、 40以上のAIエージェントスキルとMCPサーバー機能を標準搭載し、ClaudeやGemini CLIとシームレスに連携できます。 本記事では、その仕組み・インストール方法・具体的なユースケースを徹底解説します。

1. Google Workspace CLIとは？

Google Workspace CLI（GitHubリポジトリ名：googleworkspace/cli、コマンド名：gws）は、 Googleが公式に開発・公開しているオープンソースのコマンドラインインターフェースです。 Drive・Gmail・Calendar・Sheets・Docs・Chat・Adminなど、Google Workspaceのほぼすべてのサービスを 1つのコマンドツールから操作できることが最大の特徴です。

「CLIなんてエンジニアだけのもの」と思うかもしれませんが、このツールの真価はAIエージェントとの親和性にあります。 すべてのレスポンスが構造化JSONで返ってくるため、Claude・Geminiなどの大規模言語モデルが Workspaceのデータを「読んで・判断して・操作する」エージェントとして機能できるようになります。

従来のWorkspace操作
  • ブラウザで各サービスを個別に開く
  • GAS（Apps Script）の知識が必要
  • 自動化に複雑な設定が必要
  • AIとの連携は別途実装が必要
VS
Google Workspace CLI
  • ターミナル1行で全サービス操作
  • npm一発でインストール完了
  • スクリプト・CI/CDと即座に連携
  • Claude・Gemini CLIとMCP連携標準搭載

2. Discovery Serviceによる動的コマンド生成の仕組み

gwsの設計で特に革新的なのが、Google Discovery Serviceと連携した動的コマンド生成です。 多くのCLIツールは「APIが増えたら手動でコマンドを追加する」という作業が発生しますが、 gwsは実行時にGoogleのDiscovery Serviceを読み込み、 コマンドの全体構造を自動的に構築します。

つまり、GoogleがWorkspace APIに新しいエンドポイントを追加した瞬間に、 gwsも自動的にそのコマンドをサポートするようになります。 開発者がツールを更新するのを待つ必要がなく、常に最新のAPIを利用できるのです。

  • Step 01 gws コマンド実行
  • Step 02 Discovery Service読み込み
  • Step 03 コマンド体系を動的生成
  • Step 04 Google Workspace APIを呼び出し
  • Step 05 構造化JSONで結果を返す

図2：gwsのコマンド実行フロー

3. インストールと認証のセットアップ

gwsはnpmパッケージとして提供されており、Node.jsがインストールされていれば1行でセットアップできます。 なお、ソースからビルドするCargoオプションや、Nixフレークも用意されています。

手順 コマンド 説明
インストール npm install -g @googleworkspace/cli グローバルにインストール
認証セットアップ gws auth setup Google CloudプロジェクトとOAuth設定を対話的に構成
動作確認 gws drive files list --params '{"pageSize": 5}' Driveの最新5件のファイル一覧を取得

認証方式の選択肢

gwsは多様な認証方式をサポートしており、個人利用からCI/CDパイプラインまで幅広いシーンに対応しています。

認証方式 適したシーン
インタラクティブ（OSキーリング） 個人開発・日常業務での利用
手動OAuth Google Cloud Consoleから認証情報をDL
サービスアカウント サーバー間通信・自動化スクリプト
CI/ヘッドレス環境 GitHub Actions・Docker等のブラウザなし環境
既存トークン継承 gcloudコマンドのアクセストークンを流用

4. 主なコマンドと使い方

基本的な文法は gws [サービス名] [リソース] [操作] [オプション] の形式です。 --dry-run フラグで実際に変更を加えずにリクエスト内容を確認でき、 タブ補完も利用可能なため、エンジニアでなくても直感的に操作できます。

サービス コマンド例 操作内容
Drive gws drive files list --params '{"pageSize":10}' ファイル一覧を最新10件取得
Gmail gws gmail users messages list --params '{"q":"is:unread"}' 未読メール一覧を取得
Calendar gws calendar events list --params '{"calendarId":"primary"}' プライマリカレンダーのイベント一覧
Admin gws admin directory users list 組織内ユーザー一覧を取得（管理者権限）
Chat gws chat spaces list 参加中のチャットスペース一覧

5. MCPサーバーとしてClaudeやGemini CLIと連携する方法

gws最大の特徴の一つが、MCP（Model Context Protocol）サーバーとして起動できることです。 MCPはAIエージェントが外部ツールを呼び出すための標準プロトコルであり、 Claude Desktop・Gemini CLI・VS Codeなど多数のクライアントが対応しています。

以下のコマンドでMCPサーバーを起動すると、対応するAIクライアントがWorkspaceのAPIを 構造化ツールとして直接呼び出せるようになります。

コマンド 公開するサービス
gws mcp -s drive Driveのみ
gws mcp -s drive,gmail,calendar Drive・Gmail・Calendarの3サービス
gws mcp -s all サポートされる全サービス

Gemini CLIとの連携（エクステンション方式）

Gemini CLIユーザーは、専用エクステンションを1行でインストールするだけでWorkspace操作が可能になります。 認証はgwsが一括管理するため、Gemini側での追加設定は不要です。

terminal 
$ gemini extensions install https://github.com/gemini-cli-extensions/workspace

AIエージェントが実現できること

  • Step 01 Drive を検索・読み込み
  • Step 02 AI が内容を要約・分析
  • Step 03 Docs に結果を書き出し
  • Step 04 Calendar にスケジュール登録

6. 類似ツール「gogcli」との比較

gwsと並んでよく語られるのが、Peterstein Berger氏が開発したオープンソースツール 「gogcli（コマンド名：gog）」です。 2026年1月にv0.9.0がアップデートされ、注目を集めています。 両者の違いを整理します。

項目 gws（Google公式） gogcli（サードパーティ）
開発元 Google公式（googleworkspace/cli） Peter Steinberger氏（steipete/gogcli）
インストール npm install -g @googleworkspace/cli brew install steipete/tap/gogcli
カバー範囲 Discovery Service経由で全Workspace API Gmail・Calendar・Drive・Contacts・Tasks・Sheets・Docs・Slides・People
出力形式 構造化JSON（AI連携に最適） JSON・テーブル・プレーンテキスト（人間視認性が高い）
MCP対応 標準搭載（gws mcp） 非搭載
AIエージェントスキル 40以上のスキル・100以上のSKILL.mdファイル 非搭載
マルチアカウント 対応 対応（gog auth manage）
ステータス 開発中（v1.0以前・破壊的変更あり） v0.9.0（安定版）

AIエージェント連携・自動化パイプラインを構築したい場合はgws、 人間が日常業務で直感的に使いたい場合はgogcliが適していると言えるでしょう。 なおgwsはv1.0前のため、本番環境への導入は変更内容を慎重に追いながら行うことを推奨します。

7. ビジネス活用シーン3選

①　定期レポートの自動生成

毎週月曜日にGmailの特定ラベルのメールをgwsで取得し、AI（Claude・Gemini）に要約させてDocに書き出す、という処理をCron＋シェルスクリプト数行で実現できます。 従来はGASやn8nなどが必要だった処理が、CLIとAIの組み合わせでシンプルに完結します。

②　会議スケジュールの一括最適化

gws calendar events listで今週の全予定を取得し、AIに「重複・移動すべき会議」を分析させた上で、 gws calendar events updateで修正を適用するワークフローが構築できます。 複数メンバーのカレンダーを横断的に管理するシーンで特に威力を発揮します。

③　Driveドキュメントのナレッジベース化

社内Driveにあるドキュメントをgwsでバッチ取得し、ベクトルDBに格納してRAGシステムを構築する用途にも適しています。 MCPサーバー経由でAIエージェントがリアルタイムにDriveを検索・参照できる構成も可能です。

8. まとめ：CLIがAIエージェント時代の"のり"になる

Google Workspace CLIは、単なる「ブラウザ操作のCLI版」ではありません。 Discovery Serviceによる動的コマンド生成・MCP標準搭載・40以上のAIエージェントスキルという3つの特徴が組み合わさることで、 GoogleのエコシステムとAIエージェントをつなぐ"のり"として機能します。

特にClaudeやGeminiをAIエージェントとして活用する企業にとって、 gwsは「AIがWorkspaceを操作できる実装コストを劇的に下げる」ツールです。 現時点ではv1.0未満でAPIの安定性に注意が必要ですが、今後の正式リリースに向けて 今から仕組みを理解・検証しておくことをお勧めします。
